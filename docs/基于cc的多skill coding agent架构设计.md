以下是我个人对团队版CodingAgent的理解和设计

## 权限
首先会有员工个人身份信息，记录所在部门，岗位等信息，用于后面的各种鉴权 以及skill的query。
可以控制可以修改的项目目录，以及各种工具的使用限制。
比如：1.普通员工不允许使用ssh（连接生产服务器）
2.前端组不允许修改后端组的代码
3.组员的push会通过hooks自动发送给组长review。
等等。

## skill系统
### 首先我们需要沉淀skill
首先每个人可以在允许的情况下上传自己的skill到公共库里。
然后我们重点是会创建：
1.skill分级目录： 按照不同岗位，部门，具体业务和场景 来划分skill的目录。
2.skill的沉淀：如果只保存 没有评分，那么对于后面的skill选取毫无意义。所以我们会维护一个“skill使用图谱”，记录哪个部门哪个项目哪个业务哪个人哪个场景下-用了什么skill-效果如何 等等关系。
通过前端的 好/坏 反馈按钮，或者用户的脏话 来返回本次skill的使用效果。会进入异步反馈队列 异步更新当前图谱。
然后在特定时间（比如半夜） 进行聚类统计， 给skill评分。

3.skill的选择：cc的skill是不通过检索的， 但我们改良版需要进行一点检索。 通过我们的部门/团队/职位/项目/场景 来检索， 选择得分最高的topk skill。
