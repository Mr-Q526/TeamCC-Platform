# 文档管理规范

此目录（`docs/`）用于存放项目生命周期内产生的各类技术与管理文档。
请遵照以下目录结构约定分类并存放对应的文档：

## 目录结构

- **`tasks/` (任务计划)**
  - 用于存放项目功能开发、技术优化等任务的**实施计划书**（Implementation Plan）。
  - 各类重构计划、功能实现规格说明也均存放于此。
  - 创建新计划时，可参考该目录下的 `template.md`。

- **`bugs/` (bug修复)**
  - 用于记录复杂或关键缺陷的排查分析过程以及**修复方案**（Bug Fix Report）。
  - 包括从现象复现到根因分析（Root Cause Analysis）的完整记录。
  - 创建新记录时，可参考该目录下的 `template.md`。

- **`summaries/` (任务总结)**
  - 每当一个中大型任务或重构工作完成后，生成的**收尾复盘结果**或走查指南（Walkthrough / Summary）。
  - 有助于沉淀项目经验和记录遗留事项。
  - 创建新总结时，可参考该目录下的 `template.md`。

- **`architecture/` (架构设计)**
  - 用于存放系统架构设计、核心机制方案、系统改造方案等全局或深度的设计文档。

- **`reference/` (参考定义与规范)**
  - 存放开发规范、ID映射表、字段枚举字典等供日常开发查阅的标准化定义。

- **`analysis/` (研究与分析报告)**
  - 用于记录对现有产品、框架特性的深度分析、机制拆解和竞品调研等学习型文档。

---
建议所有的子文档名称遵守：`[日期]-[简短标题].md` 的格式，例如 `20260411-identity-permissions-summary.md`。
