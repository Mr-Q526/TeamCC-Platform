# 三种 Skill 检索方式评测与图谱增益分析

## 摘要

本文针对 `skill-graph` 当前的三种检索方式进行对比评测：

- `bm25`
- `bm25_vector`
- `bm25_vector_graph`

评测目标不是只验证“是否能召回某个 skill”，而是进一步验证：当多个 skill 功能接近时，图谱反馈是否能够把历史表现更优的 skill 排到更前面。为此，本文采用 `graph-preference/v1` 专项数据集，在完全相同的 case 集合上比较三种检索方式，并分别使用 `canonical` 与 `experiment` 两种 retrieval features 作为图谱输入条件。

主要结果如下：

1. 在 `canonical` 条件下，`bm25_vector_graph` 与 `bm25_vector` 指标一致，说明图谱路径已可执行，但默认正式图谱信号尚不足以产生排序增益。
2. 在 `experiment` 条件下，`bm25_vector_graph` 的 `Recall@1` 由 `0.712` 提升至 `0.852`，`MRR` 由 `0.831833` 提升至 `0.902767`。
3. 图谱加权共改写 `70` 个 case 的 Top1，且 `hurtCount = 0`，表明当前 Skill 设计体系已经具备“利用反馈持续提升正确检索率”的能力。

## 1. 研究问题

本文回答三个问题：

### 1.1 RQ1：向量召回是否优于纯 lexical 召回

即 `bm25_vector` 是否能够稳定优于 `bm25`。

### 1.2 RQ2：图谱加权是否能够进一步提升正确检索率

即 `bm25_vector_graph` 是否能够在 `bm25_vector` 的基础上，把正确 skill 更稳定地提升到 Top1。

### 1.3 RQ3：图谱增益主要出现在哪些 domain

即图谱加权的收益是全局性的，还是主要集中在具备强反馈与强场景约束的 skill 簇。

## 3. 评价指标

本文主要使用以下指标：

- `Recall@1`
  衡量目标 skill 是否出现在 Top1，可视为“正确检索率”的核心指标。
- `Recall@3`
  衡量目标 skill 是否至少进入前 3 名。
- `Recall@5`
  衡量目标 skill 是否至少进入前 5 名。
- `MRR`
  衡量目标 skill 的平均排名质量，越靠前越高。
- `NDCG@3 / NDCG@5`
  衡量整体排序质量。
- `preferredSkillTop1Rate`
  在图谱偏好专项集中，期望的优质 skill 成为 Top1 的比例。
- `preferredSkillBeatsCompetitorRate`
  期望的优质 skill 排在竞争 skill 前面的比例。
- `wrongIntentHijackRate`
  非目标且非竞争 skill 错误抢占 Top1 的比例，越低越好。

## 4. 总体结果

## 4.1 canonical 条件

| mode | Recall@1 | Recall@3 | Recall@5 | MRR | NDCG@3 | NDCG@5 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| `bm25` | 0.706 | 0.950 | 0.976 | 0.826333 | 0.832064 | 0.850154 |
| `bm25_vector` | 0.712 | 0.958 | 0.978 | 0.831833 | 0.841241 | 0.855174 |
| `bm25_vector_graph` | 0.712 | 0.958 | 0.978 | 0.831833 | 0.841241 | 0.855174 |

观察：

1. `bm25_vector` 相比 `bm25` 有小幅、稳定提升。
2. `bm25_vector_graph` 与 `bm25_vector` 完全一致。
3. 这不代表图谱路径失效，而是说明在 `canonical` 条件下，当前默认正式图谱尚未形成有效的上下文加分信号。

## 4.2 experiment 条件

| mode | Recall@1 | Recall@3 | Recall@5 | MRR | NDCG@3 | NDCG@5 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| `bm25` | 0.706 | 0.950 | 0.976 | 0.826333 | 0.832064 | 0.850154 |
| `bm25_vector` | 0.712 | 0.958 | 0.978 | 0.831833 | 0.841241 | 0.855174 |
| `bm25_vector_graph` | 0.852 | 0.960 | 0.978 | 0.902767 | 0.863640 | 0.876985 |

观察：

1. `bm25_vector` 相对 `bm25` 的提升幅度有限，但稳定存在。
2. `bm25_vector_graph` 相对 `bm25_vector` 出现显著跃升。

## 4.3 增益量化

### 4.3.1 `bm25 -> bm25_vector`

| 指标 | 增量 |
| --- | ---: |
| Recall@1 | +0.006 |
| Recall@3 | +0.008 |
| Recall@5 | +0.002 |
| MRR | +0.005500 |

结论：

- 向量召回确实优于纯 lexical，但增益属于“稳定小幅提升”。

### 4.3.2 `bm25_vector -> bm25_vector_graph (experiment)`

| 指标 | 增量 |
| --- | ---: |
| Recall@1 | +0.140 |
| Recall@3 | +0.002 |
| Recall@5 | +0.000 |
| MRR | +0.070934 |

结论：

- 图谱加权最显著的收益体现在 `Recall@1` 与 `MRR`。
- 这说明图谱不是简单增加召回覆盖，而是**把正确 skill 从已有候选中推到更靠前的位置**。

## 8. 失败分析

## 8.1 homepage 簇

homepage 相关 case 共 `20` 条：

- `bm25 Top1 = 19`
- `bm25_vector Top1 = 19`
- `bm25_vector_graph Top1 = 19`

解释：

- homepage 簇当前已经接近天花板
- 图谱加权没有明显额外收益空间
- 剩余 1 条 miss 更偏向 `intent/discriminator` 规则问题，而非反馈不足

## 8.2 ppt-maker 簇

ppt 相关 case 共 `30` 条：

- `bm25 Top1 = 14`
- `bm25_vector Top1 = 14`
- `bm25_vector_graph Top1 = 14`

主要失败原因：

1. `intent_mismatch`
2. `recall_gap_exceeded:0.15`

具体表现为：

- `design/ppt-maker` 经常被 `design/ppt-course-presentation` 抢占
- 即便图谱生效，`ppt-maker` 也因 recall gap 过大，无法进入可加分区间

结论：

- 这说明图谱加权并不能替代 recall 质量
- 对于 `ppt-maker` 这类簇，下一步应优先治理 intent 边界与相邻 skill 的语义区分

## 9. 讨论

## 9.1 这组结果说明了什么

这组结果说明：

1. Skill 系统已经不仅仅是“静态 skill 文档集合”。
2. Skill 的历史反馈已经能够进入图谱，并反向影响后续检索排序。
3. 在反馈足够、上下文可命中的条件下，图谱能够显著提升正确检索率。

更重要的是，图谱带来的收益主要体现在：

- 更高的 `Top1` 正确率
- 更高的 `MRR`

这正是实际产品体验中最重要的部分，因为用户最关心的是“系统第一名给出的 skill 是否就是对的”。

## 9.2 为什么 canonical 暂时还没有提升

当前 `canonical` 没有 uplift，原因并不在于 graph 逻辑本身，而在于：

- 真实反馈尚少
- `project / scene / department` 的覆盖不足
- 默认正式图谱尚未形成密集的 preference signal

因此：

- `canonical` 代表当前正式默认状态
- `experiment` 代表该体系在反馈充分条件下的可达上限

## 9.3 有效性边界

本文也有明确边界：

1. 当前最强结果成立于 `experiment` 条件，而非正式线上 `canonical`。
2. 数据集是图谱偏好专项集，不等同于所有真实用户任务分布。
3. 一些 skill 簇，例如 `ppt-maker`，仍然更受 recall/intent 边界制约，而非单纯受 graph 反馈制约。

因此，本文结论应理解为：

> 当前 Skill 设计体系已经被证明“有能力利用反馈提升正确检索率”，而不是说“所有 domain 已全部达到生产最优状态”。

## 10. 结论

本文最核心的结论是：

1. `bm25_vector` 已经稳定优于 `bm25`。
2. 在 `experiment` 条件下，`bm25_vector_graph` 显著优于 `bm25_vector`。
3. 图谱加权将 `Recall@1` 从 `0.712` 提升到 `0.852`，并带来 `70` 个 Top1 正向改写、`0` 个新增回退。

这说明 `skill-graph` 当前的 Skill 设计体系已经具备以下工程特征：

- 可评测
- 可积累反馈
- 可通过图谱反馈持续优化
- 可用数据证明正确检索率正在提升

如果需要向外部团队说明该体系的价值，可以用一句话概括：

> 我们已经证明，Skill 不是静态知识库，而是一个能够通过历史反馈持续改善检索质量的系统；在 500 条图谱偏好专项评测中，图谱加权将 Top1 正确率从 `0.712` 提升到了 `0.852`。
