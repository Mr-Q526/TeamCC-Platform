# 安全与协作规范提醒 (Worktree 开发需知)

> **请注意：** 这个目录是一个孤立的 Git Worktree！

在这里开发并提交 (commit) 后，请务必记得将你的改动**合并（Merge/Pull Request）回主线 `main` 分支**。

**为什么要这么做？**
1. **Docker 挂载限制：** 我们所有的基础设施容器（Postgres, Neo4j, Redis 等）为了避免端口争抢和运行环境割裂，统一放置在 `main` 目录进行 `docker compose up` 集中管理。
2. **代码隔离可见性：** 如果你的改动不合并回 `main`，其他在这台机器上通过主工作区启动服务的人，或是 Docker 挂载的本地目录，都无法看到你在此工作树中最新修改的代码！

**标准工作流：**
- ✅ 在此目录下进行日常功能开发并 `git push`（推送到本工作树对应的远端分支）。
- ✅ 开发节点完毕后，提交 Pull Request 到 `main` 并合并。
- ✅ 如果需要在全栈环境联调，请回到 `main` 目录拉取最新代码，然后再在 `main` 中启动 / 重启相关服务。
