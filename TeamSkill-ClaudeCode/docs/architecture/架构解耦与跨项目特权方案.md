# 控制平面解耦与跨项目特权架构方案 (Option B)

> 2026-04-12 更新：本文是早期草案。现行方案已统一以 TeamCC Admin 为唯一身份源；文中基于本地活动身份文件的描述不再作为实施要求。

本方案重点解决大模型 Agent 鉴权架构耦合的系统性痛点，并引入了企业级场景的“跨项目大盘”特权检索能力。

## 架构重构目标

1. **解耦策略配置 (Control Plane Decoupling)**: 移除系统针对具体业务代码库（如 `teamcc-demoproject`） `.claude/policies/*.json` 配置的硬编码依赖。将其底层加载层重构，全局通过控制面组件读写系统级注册表大盘 `~/.claude/policies`（`getClaudeConfigHomeDir`）。
2. **跨项目统管特权 (Cross-Project Capability)**: 在基于 TeamCC `IdentityEnvelope` 解析得到的 `IdentityProfile` 中保留 `projectId`（项目编号）维度。当操作角色被后台显式授予跨项目能力时，拦截引擎才允许其跨越默认项目边界；该能力不再依赖本地身份票据。

## 核心实现逻辑

### 1. `src/utils/identity.ts` 的模型升级
- 在 `IdentityProfile` 类型中声明出业务不可或缺的 `projectId: number` 字段。
- 在 TeamCC 身份加载阶段提取并严格校验 `projectId` 数据，缺失或非法时进入受限态，而不是依赖本地文件默认值。
- 抽象化跨域判权函数：增加核心鉴权特权接口 `isCrossProject(profile: IdentityProfile): boolean`（即底层判定 `projectId === 0`）。

### 2. `src/utils/permissions/identityPolicyLoader.ts` 的加载分离
- **文件读取平面分控**: 将底层硬加载逻辑 `join(cwd, '.claude', 'policies', filename)` 彻底摘除，替换基于配置模块寻址的全局中转站：`join(getClaudeConfigHomeDir(), 'policies', filename)`。
- **神权越狱判定**: 在实际组合多条规则引擎数据表前检查其身份，赋予绝对阻断的最高赦免权：
  ```typescript
  import { getIdentityProfile } from '../bootstrap/state.js'

  // 在引擎真正编译前置截流
  const profile = getIdentityProfile()
  if (profile && profile.projectId === 0) {
      logForDebugging(`[identityPolicy] Cross-project privilege detected (projectId=0). Bypassing policy restrictions.`)
      return [] // 直接跳出加载，下发清空规则
  }
  ```

## 部署实施注意事项 (Setup Sync)
为了适配控制面隔离机制，单节点开发人员需要保障本地全局配置源具有真实可测的阻断池。需要手动执行底层中转同步命令：
```bash
# 将项目内模拟的策略库上抛给本地操作系统配置大盘
mkdir -p ~/.claude/policies
cp -R .claude/policies/* ~/.claude/policies/
```
